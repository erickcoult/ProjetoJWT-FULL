Primeirta instalação

Antes de tudo, já devemos ter o NodeJS instalado na máquina para darmos inicio ao projeto. Após isso, devemos criar o arquivo de configuração json. Para isso, basta inicializar o terminal no VSCode de inserir npm init -y. Isso inicializará todos os comandos necessarios para o JSON e com o -y, aplicará yes a todas as perguntas feitas.

Instalação das Bibliotecas

Para este projeto, utilizaremos as seguintes bibliotecas: bcrypt, dotenv, express, jsonwebtoken e mongoose . Para instalar, basta apenas inserir npm intall e o nome das bibliotecas

Nodemom

Nodemon é, basicamente, um LiveServer do VSCode. Ou seja, toda modificação feita no arquivo Node, será atualizada em tempo real. Para abrirmos o servidor, primeiro instalamos o nodemon e definimos o save como dependência de desenvolvimento (npm install --save-dev nodemon)

criação Dotenv

dotenv é uma biblioteca para Node.js que carrega variáveis de ambiente de um arquivo .env para o objeto process.env. Isso permite gerenciar configurações sensíveis (como chaves de API, credenciais, etc.) de forma segura e centralizada, sem precisar definir essas variáveis diretamente no código ou no sistema operacional. Portanto, process.env é sempre global e disponível; dotenv é apenas uma ferramenta para carregar variáveis adicionais a partir de um arquivo .env específico. Para utilizarmos esta biblioteca, temos apenas que criar na pasta raiz o arquivo .env. Após isso, usamos .config() no código para que A chamada a .config() é o que efetivamente carrega as variáveis de ambiente para o process.env. Sem essa chamada, as variáveis de ambiente definidas no arquivo .env não estariam disponíveis no seu código.
* Vale lembrar que o arquivo .env deve estar incluso no ".gitignore", já que possui informações senvível que não podem ser enviadas para o repositório
